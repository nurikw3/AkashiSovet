# Руководство разработчика (AkashiSovet Bot)

Добро пожаловать в проект бота для составления служебных записок «AkashiSovet». 
Этот документ поможет быстро вникнуть в архитектуру проекта, понять как он работает и как добавлять новые функции.

---

## 📂 Структура проекта

Весь код разделен на две логические части: инициализация/конфигурация бота и основная логика (stdlib).

```
AkashiSovet/
├── .env                      # Конфиденциальные настройки и ключи (не попадает в Git)
├── app.db                    # База данных SQLite (генерируется при первом запуске)
├── uv.lock                   # Файл блокировки зависимостей (менеджер пакетов uv)
├── pyproject.toml            # Описание зависимостей и метаданных проекта
│
├── bot/                      # Ядро приложения
│   ├── bot.py                # Точка входа (отвечает за запуск и роутеры aiogram)
│   ├── config.py             # Загрузка и валидация .env через pydantic-settings
│   └── logger.py             # Настройки логирования (loguru)
│
└── stdlib/                   # Основная бизнес-логика
    ├── assets/               # Статичные файлы (логотипы для PDF)
    ├── templates/            # Текстовые шаблоны или Jinja-файлы
    ├── db.py                 # Работа с SQLite базой данных (aiosqlite)
    ├── keyboards.py          # Все кнопки Telegram (InlineKeyboardMarkup)
    ├── llm.py                # Работа с OpenAI API и Langfuse, а также кэширование
    ├── pdf.py                # Генерация PDF-документа с помощью ReportLab
    │
    └── handlers/             # Обработчики сообщений из Telegram
        ├── blocks.py         # Настройки блоков (описания, заголовки вопросов)
        ├── superuser.py      # Обработка заявок суперюзерами (согласование/отказ)
        └── user.py           # Основной флоу создания заявки пользователем
```

## 🛠 Как работает флоу пользователя?

1. **Запуск**: Пользователь нажимает `/start`. Состояние (State) устанавливается в `BotStates.FILLING`. В базе данных создается черновик заявки (`status='draft'`).
2. **Заполнение**: Пользователь отправляет текст (блок 1, 2, ...).
   - Сообщение попадает в `handle_block_input` (файл `handlers/user.py`).
   - Текст проходит **очистку** от лишних пробелов и переносов.
   - Бот собирает текст всех предыдущих блоков для контекста.
   - Бот проверяет **локальный кэш** по хешу сообщения. Если такого сообщения не было — отправляет в LLM (Groq / OpenAI API) для приведения в официально-деловой стиль.
   - Бот сохраняет ответ в кэш и переводит пользователя на этап подтверждения с помощью кнопок (дальше/изменить).
3. **Файлы**: После заполнения 5 блоков, бот просит загрузить документы или фото.
4. **Финализация**: В функции `finalize_and_notify` генерируется PDF (через `pdf.py`). Бот отправляет PDF самому пользователю и рассылает на рассмотрение всем `SUPERUSER_IDS` из `.env`.

## 🧠 Работа с LLM и Langfuse

**Langfuse** — это платформа для мониторинга, аналитики и управления промптами нейросетей. 
В нашем боте она используется для двух вещей:

1. **Трейсинг (Tracing)**: Мы логируем каждое обращение к LLM. Вызов оборачивается в контекстный менеджер `_langfuse.start_as_current_observation()`. В панели Langfuse можно увидеть, какой промпт ушел в сеть, сколько токенов было потрачено, и сколько времени занял ответ.
2. **Управление Промптами (Prompt Management)**: Бот подгружает системный промпт (инструкцию для нейросети) напрямую с серверов Langfuse. 
   - **Зачем это нужно?** Это позволяет тебе (как админу) зайти в веб-интерфейс Langfuse и изменить инструкцию для ИИ **без необходимости менять код и перезапускать бота**.

### Как менять промпт через Langfuse?
1. Зайди в дашборд Langfuse (cloud.langfuse.com).
2. Открой вкладку **Prompts**.
3. Создай или найди промпт с названием `editor_prompt`.
4. В поле "System" пропиши инструкции. В поле "User" вставь: 
   ```text
   Контекст заявки: {{context}}
   
   Текст для редактирования: {{raw}}
   ```
5. Сохрани и сделай версию **Production**. 
   *Бот будет автоматически скачивать новую версию каждые несколько минут.*

## ⚡ Кэширование (Оптимизация токенов)

Чтобы не тратить деньги (токены) на одни и те же запросы, в боте реализовано SQLite кэширование:
1. Когда пользователь присылает текст, бот чистит его от лишних пробелов (`preprocess_text` в `llm.py`).
2. Собирается массив `messages` (системный промпт + контекст + текст юзера) и превращается в уникальный хеш (SHA-256).
3. Бот проверяет таблицу `llm_cache` в базе данных `app.db`.
4. Если точно такой же текст в таком же контексте уже обрабатывался — бот моментально отдает ответ из базы. Если нет — отправляет в LLM и записывает в БД.

## ➕ Как добавить новую фичу?

### 1. Как добавить новую команду в бота (например, `/help`)
1. Открой `stdlib/handlers/user.py`.
2. Создай функцию с декоратором `@router.message(Command("help"))`:
```python
from aiogram.filters import Command

@router.message(Command("help"))
async def cmd_help(message: Message):
    await message.answer("Я бот для создания служебных записок. Нажмите /start для начала.")
```

### 2. Как добавить новый вопрос/блок в заявку
1. Открой `stdlib/handlers/blocks.py`.
2. Добавь новый пункт в словарь `BLOCKS`. Например, вопрос 6:
```python
    6: {
        "title": "Дополнительные комментарии",
        "question": "Если есть что-то еще, напишите здесь.",
    },
```
3. В `stdlib/handlers/user.py` в функции `on_confirm` измени проверку с `if current_block < 5:` на `if current_block < 6:`.
4. В `stdlib/pdf.py` добавь новый блок в список `sections`, чтобы он отображался в итоговом PDF.

### 3. Как изменить дизайн PDF
PDF генерируется программно через библиотеку **ReportLab**.
1. Открой `stdlib/pdf.py`.
2. Графические фоны лежат в `stdlib/assets/`. Они отрисовываются функцией `draw_background`.
3. Стили текста (шрифты, отступы, жирность) задаются в функции `_styles()`.
4. Сама логика построения документа находится в `generate_pdf`. Document строится как "Story" (набор блоков, которые идут сверху вниз: параграфы, отступы, линии).

## 🏃 Команды для разработки

Используй пакетный менеджер **uv** (он быстрее обычного pip):

- Запуск бота: `uv run python -m bot.bot`
- Добавление новой библиотеки: `uv add requests`
- Форматирование/проверка кода: `uv run ruff check .`
