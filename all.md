# Обзор проекта и внесённые улучшения

Документ суммирует устройство **Akashi Sovet** и ключевые технические решения (архитектура, оптимизации, безопасность, инфраструктура).

---

## Назначение

Платформа для подачи и согласования внутренних заявок (повестка Правления): **Telegram-бот** для сотрудников и **веб-панель** для администраторов. Общая бизнес-логика в `stdlib/`, доступ к данным через **PostgreSQL**, состояние и кэш — **Redis**, файлы и PDF — **S3-совместимое хранилище** (в Docker — MinIO).

---

## Архитектура (как всё связано)

1. **Бот** (`bot/`, Aiogram) ведёт пользователя по шагам заполнения заявки, FSM и обработчики в `stdlib/handlers/`.
2. **Веб** (`web/`, FastAPI + Jinja2) отдаёт админский интерфейс, скачивание PDF, правку шаблона заявки и т.д.
3. **`stdlib/db.py`** — слой **asyncpg**: заявки, пользователи, настройки, встречи и пр.
4. **`stdlib/s3.py`** — загрузка/скачивание вложений, подписей, кэшированных PDF; бакеты задаются в конфиге/compose.
5. **`stdlib/pdf.py`** — генерация служебной записки (ReportLab), единая точка для бота и веба через `get_app_pdf_buffer`.
6. **`stdlib/template.py`** — шаблон блоков заявки (Pydantic), хранение в `settings`, кэш в Redis.
7. **Планировщик** (APScheduler в боте) — фоновые задачи (отчёты, напоминания — по фактической реализации в коде).
8. **LLM** (`stdlib/llm/`) — помощь в заполнении полей через OpenAI-совместимый API (ключ и URL из `.env`).

### Время и часовой пояс

Вся пользовательская логика отображения и ввода дат/времени завязана на **Казахстан, `Asia/Almaty` (UTC+5)** через `stdlib/timezone_util.py`. В PostgreSQL хранятся **`TIMESTAMPTZ` в UTC**; формы заседаний трактуют ввод как локальное время Астаны и конвертируют в UTC перед записью.

## Оптимизации и устойчивость под нагрузкой

### Пул PostgreSQL

В `bot/config.py` введены **`DB_POOL_MIN_SIZE`** и **`DB_POOL_MAX_SIZE`** (по умолчанию 2 и 30). Пул создаётся в `stdlib/db.py` при старте. Для продакшена нужно согласовать **`max_connections`** в Postgres с верхней границей пула и числом процессов/воркеров.

### PDF без блокировки event loop

Синхронная сборка ReportLab (`doc.build`) вынесена в **`asyncio.to_thread`** через функцию `_generate_pdf_sync`: основной async-код только подгружает подпись из БД/S3 и ждёт поток.

### Кэш готового PDF (Redis + S3)

- Для заявки считается **токен версии** (SHA256): данные заявки и пользователя, ссылка на подпись, сериализованный шаблон блоков, **`updated_at`**, счётчик **`pdf:template_revision`** в Redis.
- При сохранении шаблона в админке **`persist_template`** инкрементирует этот счётчик, чтобы старые PDF не считались актуальными после смены структуры формы.
- Если Redis и S3 настроены: при совпадении токена с ключом `pdf_cache:{app_id}` PDF забирается из объекта **`pdf/{user_id}/{app_id}.pdf`**; иначе генерируется, заливается в S3, в Redis пишется токен с TTL.
- **`invalidate_pdf_cache(app_id)`** снимает ключ Redis и удаляет объект PDF в S3 (вызывается из сервиса заявок при правках данных).

### Docker и зависимости

- Сборка образа: **`uv sync --frozen`** по `pyproject.toml` и **`uv.lock`**.
- Базовый образ приложения: **Python 3.13**, чтобы совпадать с **`requires-python >= 3.13`** и избежать расхождений окружения (в т.ч. отсутствующих пакетов в контейнере).
- Один образ **`akashisovet:app`** на **web** и **bot**; сборка только у сервиса `web`, чтобы не ломать параллельный build.

## Безопасность (что заложено в коде и конфиг)

| Область | Реализация |
|--------|------------|
| Админ-сессия веба | Подписанные cookie через **`WEB_SESSION_SECRET`**; без секрета в лог пишется предупреждение (только для разработки). |
| Cookie за HTTPS | **`WEB_COOKIE_SECURE`** — включать в проде за reverse-proxy с TLS. |
| Логин в панель | **slowapi**: лимит **`WEB_LOGIN_RATE_LIMIT`** (например `10/minute`) на POST `/login` по IP. |
| Доступ админки | Список **`SUPERUSER_IDS`** в настройках. |
| Секреты | Токены и пароли только через **`.env`**, не коммитить. |
| Объектное хранилище | Ключи MinIO/S3 и пароли БД из переменных окружения. |

Дополнительно: одноразовые ссылки входа из бота (`/web`) завязаны на **`WEB_PUBLIC_URL`** и TTL токена (**`WEB_AUTH_TOKEN_TTL_SECONDS`**).

## Файлы и вложения

Вложения и подписи уходят в S3; в БД хранятся ключи/метаданные. Миграции в **`migrations/`** отражают эволюцию схемы (в т.ч. переход к S3-only для вложений — см. актуальные SQL в репозитории).

## Мониторинг
